# 【社会服务赛道】药's话 Demo——帮老人听懂药品说明书的 AI 用药安全助手

**标签：** 社会服务、社会公益

## 一、Demo 简介

**药's话（Pills Talk）** 是一个面向老年人的 AI 用药安全助手 Demo，目前以可直接打开的网页应用形式呈现，模拟微信小程序式的轻量体验。老人或家属输入药名、选择正在服用的药物后，系统会用更容易听懂的大白话解释药品用法，并提示可能存在的用药冲突。

这个 Demo 面向 60 岁以上长期服药的老年人，尤其是看不清说明书、读不懂医学术语、不熟悉手机操作，或子女不在身边的空巢老人。它不替代医生诊断，而是补上一个经常被忽略的环节：把药盒和说明书上的关键信息，转换成老人能理解、能记住、敢确认的表达。

目前 Demo 已完成以下核心功能：

| 功能 | 体验内容 |
|---|---|
| 查药百科 | 输入药名或点击常见药物标签，查看药品用途、吃法、注意事项和副作用 |
| 大白话说明 | 将“禁忌症”“不良反应”“剂量”等术语翻译成更适合老人理解的说法 |
| 药物交互排查 | 选择两种或多种药物，自动检查是否存在相互作用风险 |
| 语音对话模拟 | 通过语音入口模拟“老人直接问、AI 直接答”的使用场景 |
| 用药提醒 | 展示每日服药提醒的基础交互，帮助老人记住时间和剂量 |
| MCP 监控面板 | 点击右下角按钮，可以看到 Agent 调用工具的过程，展示“AI 不是凭空回答，而是在查工具” |

## 二、Demo 创作思路

做这个项目的起点，是一个很朴素的生活场景：很多老人并不是不重视吃药，而是药品说明书真的太难读。药名长、字很小、术语密集，同一种药单独看可能没问题，几种药放在一起吃时却可能出现风险。对年轻人来说，查一下资料、问一下医生很简单；对独居老人来说，这件事常常要等到身体不舒服后才被发现。

我希望做的不是一个“更聪明的医疗问答机器人”，而是一个能把复杂信息讲清楚的陪伴型工具。它回答的问题很具体：这个药什么时候吃？能不能和另一种药一起吃？吃完头晕是不是要注意？如果老人描述了胸痛、呼吸困难、站不稳等情况，系统应该提醒尽快联系医生或家属，而不是继续给出模糊建议。

选择 AI Agent + MCP 的原因，是因为用药安全不能只依赖大模型的“记忆”。在设计里，Agent 会先识别用户意图，再调用药品知识库、药物相互作用检测、紧急情况识别等工具，最后把结果转换成老人能听懂的表达。这样既能保留 AI 的自然语言能力，也能让回答过程更可追踪、更接近真实应用所需的安全边界。

这个 Demo 的核心判断是：AI 社会服务不一定要从宏大的问题开始。它可以先帮一个看不清字的老人，把药盒上的那行字念明白。

## 五、作品亮点

**1. 场景足够具体**

这个作品没有把“AI 医疗”做成很大的概念，而是聚焦在老人每天都会遇到的一个小问题：药到底该怎么吃。这个问题看起来小，但一旦出错，后果可能很严重。

**2. 交互尽量贴近老人**

Demo 采用大字体、清晰按钮、少步骤路径，并把医学术语翻译成“饭前至少 1 小时”“吃了可能不舒服”“如果头晕得厉害要找医生看看”这样的表达，减少老人理解负担。

**3. Agent 过程可展示**

MCP 监控面板能展示工具调用过程，让作品不只是一个 UI 壳，而是呈现“识别问题 → 调用工具 → 输出解释”的 Agent 思路。

**4. 有明确安全边界**

药's话不替代医生，不直接给出诊断结论。遇到疑似紧急情况时，它的目标是提醒用户联系医生、家属或急救，而不是让 AI 自行判断病情。
