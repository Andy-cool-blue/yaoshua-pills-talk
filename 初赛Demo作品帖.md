# 【社会服务赛道】药's话 Demo——帮老人听懂药品说明书的 AI 用药安全助手

**标签：** 社会服务、社会公益

## 一、Demo 简介

**药's话（Pills Talk）** 是一个面向老年人的 AI 用药安全助手 Demo，以可直接打开的网页应用形式呈现，模拟微信小程序式的轻量体验。老人或家属输入药名、拍照上传药盒、选择正在服用的药物后，系统会用更容易听懂的大白话解释药品用法，提示可能存在的用药冲突，并在出现紧急症状时引导联系家属或急救。

这个 Demo 面向 60 岁以上长期服药的老年人，尤其是看不清说明书、读不懂医学术语、不熟悉手机操作，或子女不在身边的空巢老人。它不替代医生诊断，而是补上一个经常被忽略的环节：把药盒和说明书上的关键信息，转换成老人能理解、能记住、敢确认的表达。

目前 Demo 已完成以下核心功能：

| 功能 | 体验内容 |
|---|---|
| 查药百科 | 输入药名或点击常见药物标签，查看药品用途、吃法、注意事项和副作用 |
| 大白话说明 | 将"禁忌症""不良反应""剂量"等术语翻译成更适合老人理解的说法 |
| 拍照识药 | 模拟拍药盒上传 → OCR 识别 → 自动返回药品说明的完整流程 |
| 药物交互排查 | 选择两种或多种药物，自动检查是否存在相互作用风险，按安全/警告/危险三级提示 |
| 症状自查 | 选择不适症状（头晕、恶心、胸痛等），系统分析并给出建议，紧急症状触发预警 |
| 语音对话模拟 | 通过语音入口模拟"老人直接问、AI 直接答"的使用场景，支持语音朗读回复 |
| 用药提醒 | 每日服药提醒，支持添加/删除/标记完成，localStorage 持久化保存 |
| 紧急求助 SOS | 左下角悬浮按钮，一键展示紧急联系人（120/家人/社区护士），模拟通知流程 |
| 字体大小调节 | A / A+ / A++ 三档字体切换，适配不同视力需求，设置自动保存 |
| 药品收藏 | 查药结果可收藏，收藏列表持久化保存，方便快速复查常用药物 |
| MCP 监控面板 | 点击右下角按钮，实时查看 Agent 调用工具的完整过程（含时间戳、参数、返回值） |

## 二、Demo 创作思路

做这个项目的起点，是一个很朴素的生活场景：很多老人并不是不重视吃药，而是药品说明书真的太难读。药名长、字很小、术语密集，同一种药单独看可能没问题，几种药放在一起吃时却可能出现风险。对年轻人来说，查一下资料、问一下医生很简单；对独居老人来说，这件事常常要等到身体不舒服后才被发现。

我希望做的不是一个"更聪明的医疗问答机器人"，而是一个能把复杂信息讲清楚的陪伴型工具。它回答的问题很具体：这个药什么时候吃？能不能和另一种药一起吃？吃完头晕是不是要注意？如果老人描述了胸痛、呼吸困难、站不稳等情况，系统应该提醒尽快联系医生或家属，而不是继续给出模糊建议。

选择 AI Agent + MCP 的原因，是因为用药安全不能只依赖大模型的"记忆"。在设计里，Agent 会先识别用户意图，再调用药品知识库、药物相互作用检测、紧急情况识别等工具，最后把结果转换成老人能听懂的表达。这样既能保留 AI 的自然语言能力，也能让回答过程更可追踪、更接近真实应用所需的安全边界。

这个 Demo 的核心判断是：AI 社会服务不一定要从宏大的问题开始。它可以先帮一个看不清字的老人，把药盒上的那行字念明白。

## 五、作品亮点

**1. 场景足够具体**

这个作品没有把"AI 医疗"做成很大的概念，而是聚焦在老人每天都会遇到的一个小问题：药到底该怎么吃。这个问题看起来小，但一旦出错，后果可能很严重。

**2. 交互尽量贴近老人**

Demo 采用大字体（支持 A/A+/A++ 三档调节）、清晰按钮、少步骤路径，并把医学术语翻译成"饭前至少 1 小时""吃了可能不舒服""如果头晕得厉害要找医生看看"这样的表达，减少老人理解负担。语音对话和拍照识药入口进一步降低了输入门槛。

**3. Agent 过程可展示**

MCP 监控面板能实时展示工具调用过程（含时间戳、参数、返回值），让作品不只是一个 UI 壳，而是呈现"识别问题 → 调用工具 → 输出解释"的 Agent 思路。每次查询、排查、症状自查都会记录完整的工具调用链。

**4. 有明确安全边界**

药's话不替代医生，不直接给出诊断结论。遇到疑似紧急情况时（如胸痛、呼吸困难、意识模糊），症状自查会触发预警并引导联系急救；左下角的 SOS 按钮提供一键紧急求助入口，模拟通知 120、家属和社区护士。

**5. 功能闭环完整**

从"查药 → 拍照识药 → 药物排查 → 症状自查 → 用药提醒 → 紧急求助"，覆盖了老人日常用药的完整场景。药品收藏和提醒数据通过 localStorage 持久化，模拟真实应用的使用连续性。
